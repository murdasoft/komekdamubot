# PLAN: Voice Transcription + UX для тупых клиентов

## Проблема (по скриншоту)

Клиенты в WhatsApp **не понимают, что нужно написать цифру 1 или 2**.
Вместо "1" они пишут: "Қазақша", "Алматы", случайные фразы.
Бот повторяет "Жазыңыз 1 (қазақша) немесе 2 (орысша)" — клиент в ступоре.

---

## ЧАСТЬ 1: UX — Текстовый бот (приоритет)

### 1.1 Умный парсинг свободного текста на шаге выбора языка

**Проблема:** Бот ждёт "1" или "2", а клиент пишет "Қазақша", "русский", "рус", "каз".

**Решение:** В `handle_whatsapp_update` на шаге `selecting_lang`:
- "қазақша", "каз", "казах", "qazaq", "kz" → выбрать язык kk (как будто нажал "1")
- "русский", "рус", "орысша", "ru", "russian" → выбрать язык ru (как будто нажал "2")
- Любой текст с казахскими символами (ә, і, ң, ғ, ү, ұ, қ, ө, һ) → kk
- Остальное → ru (большинство пользователей)

**Файл:** `app/bot/handlers.py`, строка ~1431-1449

### 1.2 Умный парсинг на шаге выбора города

**Проблема:** Клиент пишет "Алматы" вместо цифры.

**Это уже работает!** Функция `detect_city()` и `resolve_city_digit()` уже обрабатывают это.

### 1.3 Fallback с подсказкой "ты общаешься с ботом"

**Проблема:** Клиент не понимает, что перед ним бот. Отправляет произвольные сообщения.

**Решение:** Когда на шаге `selecting_lang` приходит непонятный текст (не цифра, не слово-язык):
```
🤖 Сіз чат-ботпен сөйлесудесіз / Вы общаетесь с чат-ботом.

Жазыңыз *1* (қазақша) немесе *2* (русский)
Бот тек *сандарды* түсінеді — 1, 2, 3 жазыңыз

Мысал: 1
```

### 1.4 Автоматическое определение языка + города

Если клиент пишет длинное предложение (>5 слов) на любом шаге:
1. Определить язык текста → сохранить в сессию
2. Попытаться определить город из текста
3. Если оба определены → сразу показать меню (пропустить шаги 1-2)

## ЧАСТЬ 2: Voice Transcription (русский + казахский)

### 2.1 Решение: Groq Whisper (БЕСПЛАТНО)

**Никаких серверов, никаких Docker'ов, никаких OpenAI.**

**Groq** даёт whisper-large-v3 **бесплатно**:
- ✅ 10 часов аудио / месяц (для начала хватит)
- ✅ Русский и казахский
- ✅ Быстро (whisper-large-v3, лучшее качество)
- ✅ Нет сервера, нет Docker, работает как API
- ✅ Fallback: Together AI (тоже whisper, тоже бесплатный кредит $1)

### 2.2 Текущая архитектура (без серверов)

```
WhatsApp → Green API → Vercel (serverless) → Groq (STT бесплатно)
                                            → Groq (LLM бесплатно)
                                            → Supabase (DB бесплатно)
```

**Стоимость: $0/мес** (кроме Green API ~$15/мес за WhatsApp)

### 2.3 Лимиты Groq (бесплатный план)

- **Audio:** ~10 часов/месяц, 25 MB на файл
- **LLM:** 14,400 запросов/день, 6,000 токенов/минуту
- **Модели:** whisper-large-v3 (STT), llama-3.3-70b-versatile (LLM)

Если лимиты кончатся → автоматический fallback на Together AI.

### 2.4 Fallback-цепочка (из `voice_stt.py`)

1. **Groq Whisper** — основной, бесплатный
2. **Together AI** — fallback ($1 кредит → ~50 часов аудио)
3. Оба не работают → текстовое сообщение "Напишите текстом"

## ЧАСТЬ 3: Voice → Meню маппинг (уже работает!)

Файл `app/bot/voice_router.py` уже делает:
1. STT: audio → text
2. Нормализация: "один" → "1", "ипотека" → "4"
3. Groq intent (опционально): свободная фраза → цифра меню

**Это работает!** После транскрипции голос обрабатывается как текст.

## ВАЖНЫЕ ЗАМЕТКИ

1. **Groq бесплатный** — 10 часов аудио/месяц, whisper-large-v3 (лучшее качество)
2. **Без серверов** — Vercel + Groq + Supabase = $0/мес
3. **Together AI** — fallback, $1 бесплатного кредита (~50ч аудио)
4. **Казахский язык** — Whisper его поддерживает, quality ниже чем русский
5. **Domain prompt** ("KOMEK DAMU, кредит, несие, ипотека") улучшает точность на 10-15%
6. **Если лимиты кончатся** — бот автоматически fallback'нет на Together или скажет "напишите текстом"
